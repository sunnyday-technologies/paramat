Privacy

Short version: no accounts, no cookies, no trackers, no personal data. Your shelf stays on your device and cameras are processed locally. Browser usage counters are opt-in; MCP tool calls are counted automatically only as anonymous monthly aggregates. All aggregates are public.

What stays on your device

  • My filaments (shelf) — stored in your browser's localStorage only. It is never uploaded. Export/backup is a file download you control; “Share” builds a link containing the data, which is only transmitted if you send that link to someone.
  • Barcode scan & label OCR — camera frames are processed entirely in your browser (html5-qrcode, Tesseract.js). No image or video ever leaves your device. QR codes are parsed for identity only; embedded URLs are never opened.

Browser usage counts (opt-in, off by default)

  • If you tick “Share anonymous usage counts”, opening or exporting a material sends a single event: {event, brand|product, family, goal} — nothing else.
  • We store only per-month counters per (event, material). We do not store your IP address, user agent, or any identifier, and we set no cookies. There is nothing to link events to you or to each other.
  • The entire aggregate is publicly readable — you can see exactly what we see. Untick the box to stop at any time.
  • Exception: tapping “🚩 Flag it” on a result sends the same anonymous, aggregate-only event even without the opt-in — it is an explicit action you take, and it's how we count which materials need investigating. It carries nothing personal.
  • Purpose: a neutral, cross-brand picture of which filaments the community actually uses, published openly.

Agent and MCP usage counts

  • Calls to the public ParaMat MCP tools are counted automatically as aggregate monthly events so agent usage is visible separately from browser activity.
  • MCP telemetry never stores search text, barcode values, IP addresses, user agents, MCP client names, session identifiers, or any other per-user identifier.
  • For a successful settings or barcode lookup, the counter includes the matched material identity and family. Searches and misses use generic labels only.
  • Declared synthetic/load-test calls are stored under separate test_* events and are not presented as community activity.

Hosting & third parties

  • The canonical site is static and served by Cloudflare Pages; a GitHub Pages mirror may also be available. MCP and aggregate counters run on Cloudflare Workers. These services see standard request metadata in transit under their own privacy policies; ParaMat telemetry does not persist that metadata.
  • The scanner and OCR libraries load from CDNs (unpkg, jsDelivr) only when you use those features; those requests are subject to the CDNs' policies.
  • Enhanced read (testing) downloads open OCR models (PP-OCRv4, PaddleOCR project, Apache-2.0) to your device on first use (≈16 MB, cached by your browser) plus the ONNX Runtime Web engine from jsDelivr. All recognition runs on your device — the photo never leaves it.
